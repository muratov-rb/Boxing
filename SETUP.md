# RingBornn — Setup Guide

The app **runs today with no keys** — sign-in shows a "not connected" notice and the AI uses the built-in coach engine. Follow this to switch on real accounts and Claude-powered analysis.

Everything is wired against clean interfaces, so you only paste values into `.env.local` — no code changes.

---

## 0. Install & run

```bash
npm install
npm run dev          # http://localhost:3000
```

Copy the env template and fill it in as you go:

```bash
cp .env.example .env.local
```

> After editing `.env.local`, **restart `npm run dev`** — `NEXT_PUBLIC_*` values are read at build/boot.

---

## 1. Supabase (accounts + database)

Real registration, login, logout, and email verification are powered by Supabase.

1. Create a free project at **https://supabase.com** → New project.
2. **Project Settings → API**. Copy:
   - **Project URL** → `NEXT_PUBLIC_SUPABASE_URL`
   - **anon / public** key → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
3. **Authentication → Sign In / Providers → Email**: keep **Confirm email** ON (this is the verification step).
4. **Authentication → URL Configuration**:
   - **Site URL**: `http://localhost:3000` (use your real domain in production)
   - **Redirect URLs**: add `http://localhost:3000/auth/callback`

That's enough for **email + password with verification**. Sign up → check inbox → click the link → you land authenticated on `/dashboard`.

### Google sign-in (optional)

1. In **Google Cloud Console** → APIs & Services → Credentials → **Create OAuth client ID** (Web application).
2. Authorized redirect URI: `https://<YOUR-PROJECT-REF>.supabase.co/auth/v1/callback` (shown in Supabase's Google provider page).
3. Copy the Client ID + Secret into **Supabase → Authentication → Providers → Google**, and enable it.

The "Continue with Google" button then works end-to-end.

## 2. Anthropic Claude (AI analysis)

The feasibility read, roadmap and nutrition plan are generated by Claude, with the local engine as an automatic fallback.

1. Get a key at **https://console.anthropic.com** → API keys.
2. Set `ANTHROPIC_API_KEY` in `.env.local`.
3. (Optional) `ANTHROPIC_MODEL` — defaults to `claude-opus-4-8`. Set `claude-sonnet-5` for lower cost/latency.

The key is **server-only** (used in `app/api/analyze/route.ts`); it is never exposed to the browser. If the key is missing or the call fails, the route quietly returns the local analysis so onboarding never breaks.

### Cost safety — do this so the API can never overcharge you

Anthropic billing is **prepaid, like a top-up card**: the AI can only ever spend the credit you've loaded. When it runs out, the AI features **pause and fall back** to the built-in engine — no debt, no surprise invoice, the app keeps working. To keep it that way:

1. **Start small.** Load **$5** of credit (Console → **Billing → Add credits**). That is your hard maximum until you choose to add more.
2. **Leave "Auto-reload" OFF.** In Console → **Billing**, make sure auto-reload / auto-recharge is **disabled**. This is the *only* setting that lets spend grow on its own — off means you can never spend more than you loaded. If you ever turn it on, cap it low (e.g. "$10, at most once per month").
3. **Set a monthly spend limit.** Console → **Limits → Usage limits** (or **Billing → Spend limits**) → set a hard cap like **$10/month**. A second ceiling regardless of anything else.
4. **Watch the first week.** Console → **Usage** shows your real burn rate. The app already caps how many scans/technique-reviews each user gets per day (Pro 2/day, Max 10/8), so cost per user is bounded by design — and each paying subscription (\$9.99–\$79.99) dwarfs its API cost.
5. **Optional — cut cost 40%+ instantly:** set `ANTHROPIC_MODEL=claude-sonnet-5` (or `claude-haiku-4-5` for ~5× cheaper). No code change.

With auto-reload off + a spend limit set, **$5 loaded = $5 maximum risk. Full stop.**

## 3D coach model (optional)

Lesson demos always show the 2D movement guide. If a compressed model exists at
`public/models/coach.glb`, lessons additionally show a **3D model** tab (rotate/zoom
viewer). To install one from a Tripo3D / Blender export:

1. Export a **binary GLB**: Blender → File → Export → glTF 2.0 (`.glb`), or download
   the GLB from Tripo3D directly. **Never open/save the file with a text editor and
   avoid copy tools that convert encodings** — that corrupts the binary data
   (the script detects this and refuses the file).
2. Compress + install it (targets < 10 MB from a raw scan of any size):

   ```bash
   node scripts/prepare-coach-model.mjs "C:/path/to/your-model.glb"
   ```

3. Commit `public/models/coach.glb` and push — the 3D tab appears automatically.

## Deploying

Set the same four variables in your host's environment (e.g. Vercel Project → Settings → Environment Variables), update the Supabase **Site URL** and **Redirect URLs** to your production domain (`https://yourdomain.com/auth/callback`), and redeploy.
